Setup

New programmers are encouraged to install a GNU/Linux OS as development on windows is not pleasant or predictable. Ubuntu is popular. At least one person has had success using virtual box on windows and passing through the prolific The following packages are recommended:

  • gcc-arm-none-eabi
  • build-essential
  • git
  • Some kind of editor (vim, gvim, gedit etc.)
  • open-jdk (java, for running the uploader

Once you have this set up you can run:

git clone https://github.com/liberty-university-robotics-club/vex.git #clone the git repo to your computer cd turing #enter the repo make #build the firmware make upload #upload the firmware to the robot (you may need to be root/in the dialout group)

How we use git

You should never commit to the master branch. Before making changes use git branch somename git checkout somename To create a new branch and switch to it, commit often and then use git push origin somename To push it to the repo. Later we will merge it into master. This prevents people from stepping on each other.

Code organization

tank.c and tank.h contain dead reconning code, read tank.h for an idea of how the interface works. controlloop.h and controlloop.c contain generalized control loop code. opcontroll.c and auto.c contain the main loops for the auto and operator parts of the game respectively.

There is no wire diagram for the robot, the include file "robot.h" serves as both the wiring diagram and the pin definitions for the firmware. Changes to sensors and motors go there first and then the hardware is modified to match it. When in doubt read robot.h. It also contains a few constants such as the speed for various motors. Function prototypes do not belong there.

The sim directory contains some test harnesses. James said he got it working last year. build.sh and test.sh are used for building and running the simulation

There are some binary files from PROS (an archive file and a java upload tool) along with linker scripts that are needed from the firmware folder. Don't touch those.
